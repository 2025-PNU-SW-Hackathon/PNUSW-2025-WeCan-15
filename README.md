# WeCam 프로젝트 소개

## 1. 프로젝트 소개

### 1.1. 개발 배경 및 필요성

- 창업 아이디어의 필요성과 사회적 의미

  현재 대학 학생회는 공지 전달, 예결산안 관리, 회의록 작성 등 다양한 업무를 여러 도구에 분산하여 처리하고 있습니다. 이로 인해 업무 누락이 빈번하고, 매 학기 경험이 부족한 새 임원들이 인수인계를 받으며 구조적 비효율이 반복되는 문제가 발생합니다. 이는 학생 자치 기구에 대한 신뢰도 저하와 학생 사회의 연결 단절로까지 이어지고 있습니다. 본 아이디어 기획팀 역시 학생회 임원 활동을 통해 이러한 문제를 직접 경험했습니다. 따라서 학생회 업무를 자동화하고 표준화하는 통합 플랫폼은 단순한 편의를 넘어, 학생 자치의 지속 가능성을 높이는 핵심적인 해결책이 될 수 있습니다.

- 국내외 시장에서의 이점 및 창업 아이디어의 사회적 가치

  국내외 대학 대부분은 여러 범용 서비스를 조합하여 사용하고 있어 통합된 학생회 전용 플랫폼에 대한 수요가 분명히 존재합니다. 위캠퍼스는 학생회 업무 흐름에 맞춰 소속 인증, 예결산 자동화, 공지 승인 등 특화된 기능을 제공하여 기존 솔루션의 한계를 극복합니다. 이는 단순한 협업 도구를 넘어 학생 자치의 투명성과 책임성을 보장하는 사회적 인프라로 기능할 수 있습니다. 위캠퍼스는 디지털 전환이 미치지 못했던 학생 자치 영역의 공백을 메우고, 학생 사회의 자율성과 지속성을 강화하는 사회적 가치를 창출하고자 합니다.


### 1.2. 개발 목표 및 주요 내용

* **목표**  
  → 학생회 중심의 **체계적인 대학 플랫폼 구축**

* **주요 기능** 
  * AI/OCR 기반 소속 인증 시스템: 재학증명서 등을 분석해 위조 여부를 검증하고 자동으로 소속을 분류합니다.
  * 단체방 자동 생성 및 관리: 인증된 구성원을 바탕으로 단체방을 자동으로 생성하여 수작업을 없앱니다.
  * 계층적 공지사항 전달: 총학생회부터 학과까지 조직 구조에 따라 공지를 일괄적으로 전파하여 누락을 방지합니다.
  * 회의록 관리: 회의록 작성과 공유, 보관을 용이하게 합니다.
  * 중앙 자료보관소: 과거 자료, 제휴 업체 목록 등을 안전하게 보관하고 인수인계 시 활용합니다.


### 1.3. 세부 내용

* **조직 구조**  
  → 총학생회 → 단과대학 학생회 → 학과 학생회  
    (계층형 워크스페이스 구성)

* **가입 및 승인**
  * 학생회는 승인 후 워크스페이스 생성
  * 일반 학생은 재학증명서 기반 AI/OCR 인증 시스템을 통해 가입 및 자동 소속 분류 


### 1.4. 기존 서비스 대비 차별성

* **학생회 중심 설계**: 기업용 범용 툴(카카오워크, Google Workspace)과 달리 학생회 실제 업무에 불필요한 기능은 제외하고 필수 기능만 통합했습니다.
* **학교 조직 체계 자동 구성**: 증명서 기반 인증을 통해 개인정보 수집 없이 조직과 단체방을 자동으로 구성합니다.
* **학생 대상 공지/채팅 연동**: 학생회 업무와 일반 학생과의 소통 채널을 단일 플랫폼으로 통합하여 정보 분산을 막습니다.
* **학생회 교체 시 인수인계 기능 내장**: 업무 프로세스와 자료가 시스템에 기록되어 임원진이 교체되어도 조직의 연속성을 보장합니다.
* **현장 경험 기반 설계**: 개발팀 대부분이 학생회 임원 출신으로, 실제 사용자 관점에서 직관적이고 실용적인 기능을 설계했습니다.


### 1.5. 사회적 가치 도입 계획

* **학생회 운영 문화 개선**: 업무 자동화와 표준화된 프로세스를 통해 학생 리더들의 행정 부담을 줄이고, 회계 투명성을 높여 학생 자치에 대한 신뢰를 회복합니다.
* **학생 참여 확대**: 공지, 알림, 설문 기능으로 일반 학생들의 학생 자치 참여를 유도하고 정보 격차를 해소합니다.
* **교육 격차 해소 및 지역 상생**: 예산이나 인력이 부족한 지방 및 소규모 학생회에도 동일한 수준의 디지털 행정 환경을 제공합니다. 또한, 플랫폼 내에서 지역 소상공인과의 제휴를 연계하여 지역 경제 활성화에 기여할 수 있습니다.
* **청년 고용 기회 창출**: 기업 채용 공고나 캠퍼스 리크루팅 연계를 통해 청년 고용 촉진에 긍정적인 영향을 미칠 수 있습니다.

### 3.4. 디렉토리 구조

📁 domain-common
공통 도메인 엔티티 및 enum 관리 모듈

src/main/java/org/example/model

affiliation: 소속 인증 관련 엔티티 (AffiliationCertification, AffiliationFile 등)

common: BaseTimeEntity 등 공통 엔티티 상속 구조

council: 학생회, 부서, 멤버, 권한 정책 엔티티

enums: 시스템 전반에서 사용하는 열거형 모음 (권한, 상태, 분류 등)

invitation: 초대 코드 및 기록 엔티티

organization: 조직 및 조직 생성 요청 관련 엔티티

todo: 할일(Todo) 관련 엔티티

user: 유저 정보, 가입 이력, 개인정보 등

📁 wecam-backend
일반 사용자 + 클라이언트용 API 서버

src/main/java/org/example/wecambackend

WecamBackendApplication.java: 백엔드 메인 실행 클래스

common: 예외 처리, 응답 포맷, 컨텍스트 관리 등 공통 유틸

config: Spring Security, JWT, Redis, OpenAPI 등 설정

controller: API 엔드포인트 컨트롤러

admin: 관리자 페이지용 API

client: 일반 사용자(학생)용 API

publicinfo: 공개 API (학교/조직 조회 등)

dto: 요청/응답 DTO

repos: JPA Repository

service: 서비스 로직 계층

util: 유틸 클래스 (현재 유저 정보, 암호화 등)

📁 wecamadminbackend
관리자 페이지 전용 백엔드 서버 (Spring + Thymeleaf 기반)

src/main/java/org/example/wecamadminbackend

WecamadminbackendApplication.java: 관리자 백엔드 메인 실행 클래스

controller: 관리자 뷰 라우터 (dashboard, organization, login)

dto: 관리자 뷰에서 사용하는 DTO

entity: 관리자용 유저 엔티티 (ex. AdminUser)

repos: 관리자 전용 JPA Repository

service: 관리자 기능 구현 로직 (조직 승인 등)

util: 암호화, 패스워드 해싱 유틸리티
